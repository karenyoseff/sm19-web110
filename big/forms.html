Social Media Optimization (SMO)

Forms are tables that users can fill out to send data back to the server. This contains normal content, markup, special elements called controls and labels, after the user is done filling or selecting the form, they complete it by submitting the form to an agent (mail, web server). “The form is then usually submitted to the website and the data is used in some way. Your customer details might be stored in a database for later use, or the website might be updated after you choose a product to view.” 1 With this you need to set up a web server ro process and receive the data.

A HTML form contains one or more widgets, like text fields, select boxes, buttons, checkboxes, or radio buttons, and all of these are related with a label that explains the purpose and the instruction could be clear.

Before starting your code, it is important to plan your form. It is important to remember that if you don’t plan it ahead or if you make a big form you might have the risk to lose users. So it is important to make a useful form that can collect all the data you need and at the same time fast to fill up.

To create a form on HTML you need the elements: <form> </form>, <label>, <input>, <textarea> and <button>. To name the data in a form you need to use the name attribute on each form widget that will collect a specific piece of data.
