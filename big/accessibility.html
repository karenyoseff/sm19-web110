Accesibility

Web Accessibility covers a wide range of features that a web site should have in order for a web site to be usable by everyone:

- Color blindness – We have to be careful not to use colors on our website that are hard to distinguish for people with color blindness, those are mostly red and green hues but there are others that we should be careful about.

- Poor eyesight – When designing a web site we should not use a really small font, it could be tempting to fit a lot of text into a small area or use colors that look good but they are not highly contrasting, if we choose to follow any of the above points we would be affecting people with poor eyesight making our web site not accessible to them.

- Blindness – We need to make sure that all our image tags contain an alt text that can be read by a screen narrator software, if an image is not important we can add a blank space in the al text property, also we don’t need to make our web site image dependent, for example having an image of a text that a screen narrator won’t be able to read.

- Deaf – Don’t make our pages dependent on sounds that will be the only way for users to navigate through it, the web page needs to provide either explicit text indications or tooltips.

It’s also important to mention that a mailbox for reporting accessibility issues could be pretty helpful so you can add it to your web page help section and users can report accessibility related issues directly there so those issues don’t get lost among the other issues or questions users could have.

When developing a web page we need to start thinking about accessibility on the early stages of it and also throughout the whole development cycle so we can detect problems and address them early on the process this way it will be easier to address them, a simple way of testing this out could be as simple as changing the computer to high contrast mode to check if the web page will be readable.

“There are evaluation tools that help with evaluation. However, no tool alone can determine if a site meets accessibility guidelines. Knowledgeable human evaluation is required to determine if a site is accessible.” 1
